Renumbering. Click a section to expand. Select an option and click execute.

Object Renumber
Explanation of Options

1. Create initial CSV File: This option generates a CSV file containing existing object IDs and names, preparing it for renumbering. You can then manually edit this file to specify the new IDs. This is the first step in the renumbering process.

2. Open CSV File and fill new ids manually: This option allows you to open the previously created CSV file and manually enter the desired new IDs for each object.

3. Renumber objects from CSV File: This option executes the renumbering process based on the new IDs provided in the CSV file. The objects in your project will be updated with the new IDs specified.

Other Actions
Explanation of Options

1. Generate Empty Objects: Generates empty "logic" tables (copies of existing tables without internal code) and associated data transfer codeunits. You'll need to: 1) Specify a number range for the new object IDs and 2) Choose a target folder where the new objects will be created.

2. Generate Empty C/SIDE Objects: Generates empty "logic" C/AL table objects in a single target text file. You'll need to choose the target text file.

3. Insert Fields C/SIDE: JAM Insert extension fields into an existing C/SIDE table definition text file. You'll need to choose the target text file.

4. Update previous CSV: JAM Update previous CSV Renumbering file with new objects in the workspace if changed from the time file was generated. You must choose a csv file.
